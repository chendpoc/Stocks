# T006: LangGraph Stage 1 Runtime + Decision/Outcome/Evaluation/Insight v0

## Supersession

This task supersedes the old `model-decision-store` T006 plan. The old plan only covered:

```text
DecisionEnvelope -> model_decisions -> decision_feature_snapshots -> CLI/TUI read-only review
```

The confirmed Stage 1 scope is now:

```text
apps/trader-workflows
  -> DecisionGraph
  -> OutcomeGraph
  -> EvaluationGraph
  -> InsightExplorationGraph v0
```

TUI, paper execution, broker mirror, automatic training, and automatic promotion are deferred.

## Goal

Build the minimum durable self-evolving trading-agent loop:

```text
Raw Evidence
-> WeightedContextItem
-> immutable ContextSnapshot
-> DecisionGraph
-> model_decisions
-> OutcomeGraph
-> decision_outcomes
-> EvaluationGraph
-> promotion recommendation
```

Also add `InsightExplorationGraph v0` for controlled ReAct exploration that persists only `InsightCandidate`.

Runtime placement:

```text
apps/trader-workflows  LangGraph runtime + checkpoint store + graph implementation
apps/trader-agent      backend domain schema/API only
apps/trader-cli        thin command wrappers only
```

Frozen implementation contracts:

- CLI wrappers call `apps/trader-workflows` through `npm --prefix apps/trader-workflows run workflows -- <command> --json`; they do not import graph/runtime code.
- Runtime checkpoint state defaults to `data/trader-workflows/checkpoints.sqlite` and may be overridden by `TRADER_WORKFLOWS_CHECKPOINT_DB`.
- S1 uses `@langchain/langgraph` plus a project-owned `Stage1CheckpointStore` SQLite facade backed by `better-sqlite3`.
- Stage 1 domain API lives under `/api/intel/stage1` with explicit create/list/detail routes and immutable/idempotent writes.
- DecisionGraph pre-creates `decision_outcomes` pending rows; OutcomeGraph only finalizes due pending rows.
- `apps/trader-workflows` owns `src/api/client.ts` and `src/llm/provider.ts`; CLI modules are reference-only patterns, not imports.

## Boundaries

Allowed:

- LangGraph minimal durable runtime.
- New in-repo `apps/trader-workflows` app.
- Separate LangGraph checkpoint store.
- Minimal domain tables in `market_intel.db`.
- CLI commands and markdown/json reports.
- Controlled ReAct that only writes `InsightCandidate`.
- HumanOverride with immutable original DecisionEnvelope and dual-track evaluation.

Forbidden:

- Self-built TUI pages.
- Paper order submit/query/cancel.
- Broker Mirror or Reconciler.
- Automatic model promotion or model switching.
- Automatic model training.
- Full Model Registry.
- Full multimodal intelligence schema.
- Rewriting historical ContextSnapshot weights.
- LLM-only promotion from InsightCandidate to AcceptedLesson.
- Legacy hypotheses/predictions dual-write.
