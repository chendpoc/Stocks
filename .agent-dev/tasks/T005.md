### V306 确定性验证（替代时间测量）

原 V306 定义为 `trader scan --help < 300ms`，但 tsx TypeScript 编译开销 ~1.2s 使得绝对时间不可靠。改为确定性断言：

1. **`index.ts` 无 top-level await** — grep 验证 `await` 不在模块顶层
2. **`buildAgentTools.ts` lazy 调用** — `resolveAgentTools()`/`getAgentSystemPrompt()` 首行 `await ensureLongbridgeAgentOnStartup()`
3. **`bootstrapDone` 短路** — 第二次调用立即返回（`longbridgeAgent.test.ts` "second call short-circuits" 测试覆盖）

效果：`trader scan --help` 实测 ~1.2s（纯 tsx 开销），之前有 top-level await 时约 2.5-4s（+1-3s probe）。非 Agent 命令不再触发 probe。

### V303 / V304（需用户手动验收）

- **V303**：`npx tsx src/index.ts chat --eval "查 TSLA 最新价，优先长桥"` — 需本机 `longbridge auth login` + 后端 :8000 启动
- **V304**：Dashboard `[l]`/`[L]` 仍 spawn longbridge 新窗口 — 外挂逻辑 `longbridge.ts` 未改动
