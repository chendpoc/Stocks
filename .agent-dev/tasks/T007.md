# T007: LangGraph-Native DecisionGraph + Studio Adapter

## Goal

Refactor the T006 `DecisionGraph` from a hand-written class flow into a real LangGraph-native `StateGraph`, expose it through LangGraph Studio, and preserve the existing CLI JSON contract.

T007 is not a new app and not a custom UI task. It stays inside:

```text
apps/trader-workflows
  native DecisionGraph + LangGraph Studio config + checkpointer integration

apps/trader-cli
  thin wrapper behavior preserved
```

## Why This Exists

T006 added `apps/trader-workflows`, but the current `DecisionGraph` is still implemented as a normal class. `Stage1Runtime` wraps that class in a generic one-node `StateGraph`, which does not give useful Studio visibility into the business nodes and creates an ambiguous checkpoint boundary.

T007 makes the business workflow itself native:

```text
normalize_input
-> build_context_snapshot
-> generate_decision_envelope
-> validate_decision_envelope
-> persist_model_decision
-> schedule_model_path_outcomes
-> final_output
```

## Confirmed Scope

Allowed:

- Add `apps/trader-workflows/langgraph.json`.
- Add LangGraph Studio dev dependency/script.
- Add official LangGraph SQLite checkpointer dependency or wrapper.
- Refactor only `apps/trader-workflows/src/graphs/decisionGraph.ts` and directly related graph state/node/test files.
- Keep `runDecisionGraph(input)` as the adapter.
- Preserve `trader decide SYMBOL --json`.
- Preserve `trader runs list/show/resume`.
- Downgrade `Stage1CheckpointStore` to run registry and metadata storage.
- Use `thread_id = run_id` for native `DecisionGraph`.
- Map `run_id` to checkpoint metadata.
- Keep graph state bounded to processed context and refs.
- Allow Studio load-only smoke and optional fixture invocation.

Forbidden:

- No custom workflow UI.
- No React Flow editor.
- No evidence detail UI.
- No raw evidence large-object display.
- No Studio direct real-persistence.
- No backend schema/API changes.
- No `OutcomeGraph` native migration.
- No `EvaluationGraph` native migration.
- No `InsightExplorationGraph` native migration.
- No paper execution.
- No broker mirror.
- No model training.
- No automatic model promotion.
- No `apps/trader-cockpit` changes.

## Contract Summary

- `apps/trader-workflows/langgraph.json` uses `dependencies: ["."]`, `graphs.decision_graph`, and `env: "../../.env"`.
- `decision_graph` points to the exported compiled graph in `apps/trader-workflows/src/graphs/decisionGraph.ts`.
- `runDecisionGraph` invokes the compiled graph and returns the existing `DecisionGraphResult`.
- CLI keeps `run_id` as the user-facing id.
- LangGraph uses `thread_id = run_id` and checkpoint metadata internally.
- `Stage1CheckpointStore` stores run metadata and summaries, not full native graph state checkpoints.
- `runs show RUN_ID --json` returns bounded metadata and summaries.
- Studio direct real-persistence is forbidden in T007.
- Graph state stores processed context, weighted items, and `evidence_refs`, not raw evidence objects.
